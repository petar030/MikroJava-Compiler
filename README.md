# MicroJava Compiler

A complete compiler for the **MicroJava** language, developed for the course **Programming Languages Translators 1** at the School of Electrical Engineering, University of Belgrade.

## Quick Example Program

```mj
program example302

const int one = 1;
int x;

{
  int sum(int a, int b)
  {
    return a + b;
  }

  int inc(int value)
  {
    return sum(value, one);
  }

  void main()
  {
    x = inc(2);
    if(x == 3) print('O');
    else print('X');
    print(eol);
  }
}
```

This project implements the full compilation pipeline:
- lexical analysis (scanner)
- syntax analysis with AST construction
- semantic analysis with symbol table checks
- bytecode generation for the MicroJava Virtual Machine (MJ VM)

## Supported Language Features

- Global and local variables
- Arrays
- Enum types
- Methods/functions with parameters and return values
- `if-else` statements
- `for` loops (`break` / `continue`)
- `switch` statements
- Ternary operator (`?:`)
- Logical operators with short-circuit behavior (`&&`, `||`)

## Project Structure

- `spec/` – lexer/parser specifications (`mjlexer.flex`, `mjparser.cup`)
- `src/` – generated and handwritten compiler sources (`Yylex`, `MJParser`, AST, semantic/codegen classes)
- `test/` – MicroJava test programs and generated `.obj/.out/.err` files
- `lib/` – required dependencies (JFlex, CUP, symbol table/runtime jars, Log4j)
- `build.xml` – Ant build script

## Main Components

- **Compiler**: Orchestrates the complete pipeline (lexing → parsing → semantic analysis → code generation).
- **SemanticAnalyzer**: Performs semantic checks (types, scopes, method calls, `main` validation, etc.).
- **CodeGenerator**: Emits MJ VM bytecode and manages control-flow jump addresses.
- **Log4JUtils**: Handles Log4j setup and log file preparation.

## Requirements

- Java JDK (recommended: Java 8+)
- Apache Ant
- Included jars in `lib/`:
  - `JFlex.jar`
  - `cup_v10k.jar`
  - `symboltable.jar`
  - `mj-runtime.jar`
  - `log4j-1.2.17.jar`

## Build the Compiler

From the project root:

```bash
ant lexerGen
ant parserGen
ant compile
```

What these steps do:
- `lexerGen` runs JFlex over `spec/mjlexer.flex`
- `parserGen` runs CUP over `spec/mjparser.cup` and generates parser + AST classes
- `compile` compiles Java sources

## Compile a MicroJava Program

The current `Compiler` entry point compiles `test/program.mj` and generates `test/program.obj`.

Run the compiler main class from your IDE (or equivalent Java command), using:
- class: `rs.ac.bg.etf.pp1.Compiler`

> Note: In this repository, `Compiler.java` is located under `test/rs/ac/bg/etf/pp1/` and currently uses a fixed program name (`program`).

## Run Generated Bytecode

### Run using Ant target

```bash
ant run
```

By default, `run` executes `test/${test.name}.obj` where `test.name` is defined in `build.xml` (currently `program`).

### Change the executed test program

You can edit `test.name` in `build.xml`, for example:
- `program`
- `test301`
- `test302`

Then run again:

```bash
ant run
```

## Disassembly and Debug Execution

```bash
ant debugObj
ant runDebug
```

- `debugObj` disassembles `test/program.obj` and runs it in VM debug mode
- `runDebug` runs it after the debug/disassembly step

## Included Test Programs

- `test/test301.mj`:
  - Basic types, enum declarations, arrays
  - Arithmetic and increment operations
  - Array indexing and output printing

- `test/test302.mj`:
  - More complex control flow

## Logging

Log configuration is provided via Log4j files in `config/` (and related runtime configs), while generated logs are written to `logs/`.
